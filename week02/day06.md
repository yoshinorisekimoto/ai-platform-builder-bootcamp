# Day 06 - AI-Assisted API Troubleshooting

## Today's Topic
- AI-assisted API troubleshooting
- Comparing AI roles in debugging

## Fact
I asked ChatGPT, Claude, Codex, and Perplexity the same API troubleshooting question about a 401 Unauthorized error.
Although the prompt was identical, each AI approached the problem differently.

## Insight
The experiment showed that the biggest difference was not answer quality, but role.
- ChatGPT acted like a coach, providing a balanced troubleshooting approach.
- Claude behaved like a teacher, offering detailed explanations and technical reasoning.
- Codex responded like a senior software engineer, focusing on practical implementation and systematic debugging.
- Perplexity acted as a researcher, providing source-backed information and references.

Choosing the right AI depends on the role you need, not simply the tool itself.

## Connection to My Experience
In my work, API troubleshooting requires both technical investigation and clear communication with partners.
Different AI tools can support different parts of that workflow, from research and explanation to implementation and debugging.

## Next
Experiment with assigning different roles to the same AI and observe how its responses change.
