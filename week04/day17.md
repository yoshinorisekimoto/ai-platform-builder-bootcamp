# Day17 - Local Development Environment

## Goal
Build a local AI-assisted development environment using:
- VS Code
- GitHub Desktop
- Git
- Codex CLI

---

## Why Local Development?

Working locally provides:

- Faster iteration
- Better Git integration
- Version control
- Easier debugging
- Reproducible development

---

## Components

| Tool | Purpose |
|------|---------|
| VS Code | Code editor |
| Git | Version control |
| GitHub Desktop | Git GUI |
| Codex CLI | AI coding assistant |

---

## Workflow

1. Open project
2. Create feature
3. Ask Codex
4. Review changes
5. Commit
6. Push

---

## What I Built

Today I built a local AI-assisted development environment using VS Code, Git, GitHub Desktop, and Codex CLI. I also learned how to manage files with GitHub Desktop and GitHub Web.

---

## Explain Like I'm 13

Think of your computer as your own workshop.

- VS Code is your workbench.
- Git remembers every change you make.
- GitHub stores your work online.
- GitHub Desktop helps you use Git without typing many commands.
- Codex acts like an AI teammate that helps you write and review code.

Together, they create a safe environment for building software.

---

## What Humans Should Check

- Is the project stored in the correct folder?
- Are files committed before pushing?
- Is GitHub showing the expected file structure?
- Did I accidentally create or move files to the wrong location?

---

## Platform Builder View

A reliable development environment is the foundation of every AI platform project.

Instead of focusing only on coding, platform builders also care about version control, reproducibility, collaboration, and safe deployment workflows.
